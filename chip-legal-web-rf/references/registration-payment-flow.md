# Registration and payment flow

## Registration

Minimum generic shape:

1. User enters email/phone/account.
2. User accepts privacy/personal-data terms when needed.
3. User accepts service rules if access to community/account depends on them.
4. User may optionally opt in to advertising/newsletters.
5. System stores consent evidence.

## Payment

Minimum generic shape:

1. User sees price, product, payment period, and included access.
2. User accepts the public offer.
3. User accepts data-processing terms tied to payment/access.
4. If recurring charges exist, user separately accepts recurring charge terms: amount, period, next charge, cancellation route.
5. Optional advertising consent stays optional.
6. System stores document versions and consent evidence with order/payment id.

## Copy rule

Do not write “By continuing, you agree to everything” when separate consent is required. Use explicit labels with links.
