# Standard legal pack map

## Core public website pack

| Document | Purpose | Usually linked from |
|---|---|---|
| Public offer / terms | Contract terms, acceptance, price/payment, service delivery, refunds, limits | footer, checkout, pricing |
| Privacy and personal-data policy | Personal-data categories, purposes, processors, rights, storage, security | footer, registration, checkout |
| Personal-data consent | Active consent to process personal data where consent is used as legal basis | registration, checkout, lead forms |
| Cookie/analytics consent or notice | Technical identifiers, analytics, preference cookies | cookie banner, footer |
| Advertising/newsletter consent | Marketing messages and promos | newsletter opt-in, checkout optional box |
| Service/community rules | Conduct, access limits, moderation, content rules | registration, member area |

## Conditional documents

| Condition | Add |
|---|---|
| Paid trial or recurring charges | Autopay / recurring-payment consent |
| Product-specific risky service | Product addendum with exact scope and limitations |
| Experts, interviews, recordings | Expert/content release |
| User-generated public content | Publication/license terms and moderation rules |
| Children/minors | Separate age and guardian/legal review path |
| Medical/financial/legal/education-sensitive claims | Strong disclaimers and lawyer review |

## Versioning

Every document used in checkboxes should have:

- stable URL;
- version/date;
- immutable snapshot or stored text hash;
- changelog policy;
- migration plan for users who accepted an older version.
